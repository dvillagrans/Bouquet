{\fontsize{12}{15}\selectfont\bfseries RESUMEN\par}
\end{center}

\vspace{0.3cm}
\noindent El presente sistema es una plataforma digital de gesti\'on restaurantera dise\~nada para
modernizar el flujo de operaci\'on en cadenas de restaurantes con estructura jer\'arquica
multinivel (cadena, zona y sucursal). El sistema permite la gesti\'on completa de mesas
virtuales activadas mediante c\'odigo QR, el procesamiento de \'ordenes individuales y
compartidas entre comensales, el registro del estado de liquidaci\'on de cuentas de forma
trazable y el monitoreo en tiempo real de la operaci\'on por parte del personal colaborador.
La plataforma contempla cuatro roles de empleado (Administrador, Gerente, Mesero y Chef)
y dos roles de consumidor (Anfitri\'on y Comensal), cada uno con permisos diferenciados
sobre los m\'odulos del sistema. La arquitectura t\'ecnica se apoya en Next.js con Server
Actions, PostgreSQL como base de datos principal v\'ia Prisma, y Supabase Realtime para
la sincronizaci\'on en tiempo real de \'ordenes, estados de mesa y liquidaciones.
Complementariamente, se implementa una arquitectura Medallion de procesamiento de datos
mediante Apache Spark para la generaci\'on de m\'etricas anal\'iticas, velocidades de venta
y pron\'osticos de demanda, garantizando la separaci\'on entre las cargas transaccionales
y las consultas anal\'iticas complejas.

\vspace{0.3cm}
\noindent\textbf{Palabras clave:} Aplicaciones Progresivas; Restaurantes;
Sistemas Distribuidos; Inteligencia de Negocios; Bases de Datos.
